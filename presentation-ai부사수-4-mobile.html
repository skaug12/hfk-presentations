News 1 · Claude Opus 4.7

"한 번에 책 한 권을 읽고, 사진의 픽셀까지 본다"

2026년 4월 공개된 새 모델 Claude Opus 4.7. 한 번에 다룰 수 있는 맥락이 1M 토큰(≈책 한 권)으로 늘고, 이미지도 3.75MP 고해상도까지 바로 읽습니다.

무엇이 달라졌나

  • 1M 토큰 맥락 — 녹취록 + 자료 + 템플릿 한 번에
  • 2576px / 3.75MP — 기존 1568px / 1.15MP에서 대폭 상향
  • 이미지 좌표 1:1 매핑 — 픽셀 단위로 위치 지목 가능

Claude Code에서 바로 활용

모델 전환은 /model 한 줄. 무거운 작업은 4.7, 단순 변환은 Sonnet/Haiku로 배치.

HFK 기준 체감 포인트

  • 후기 생성 — 녹취 원문을 그대로 읽고 발언을 놓치지 않음
  • 시즌 레터 — 이전 6호치를 전부 참고해 톤 일치
  • 이미지 분류 — 카드뉴스 여백/정렬까지 판단

한 줄 요약 — "원문을 잘라서 넣을 필요가 없어졌다."

"요약부터 맡기던 시대가 끝난다"

전에는 이랬습니다

  • 녹취 60분짜리를 먼저 요약해서 넣기
  • 템플릿·레퍼런스 골라서 넣기
  • 이미지는 작게 줄여서 넣기

"자르는 것" 자체가 시간이었고, 맥락이 빠짐.

지금은 이렇게 됩니다

  • 녹취 원문 + 이전 후기 10편 + 템플릿을 통째로
  • AI가 직접 "중요한 발언"을 판단
  • 카드뉴스 PNG 원본을 그대로 읽고 조정 제안

두 가지 시사점

  1. "요약"이 사람의 일이 아니게 됨 — 원문 통째로 + 관점 지정이 더 정확함.
  2. "이미지 설명 능력"이 의미를 잃음 — AI가 픽셀을 직접 보니 말로 풀 필요 없음.

우리에게 주는 신호 — "AI에게 좋게 잘라주는 일"이 줄어드는 중. 우리가 해야 할 건 관점을 명확히 주는 것.

News 2 · Claude Code × Figma

디자인 도구와 AI가 왕복하는 시대

피그마가 Figma MCP 서버를 공개. 클로드 코드가 피그마 파일을 직접 읽고, 반대로 클로드 코드로 만든 UI를 피그마 프레임으로 보낼 수 있게 됨.

두 방향 모두 가능

  • Figma → Code — 프레임 URL을 붙이면 작동하는 코드 생성
  • Code → Figma — VS Code에서 만든 UI를 프레임으로 내보내기
  • 디자이너는 피그마, 개발자는 코드로 같은 결과물을 다룸

MCP가 뭔지 한 줄로

AI가 외부 도구(피그마·슬랙·구글)를 통일된 방식으로 조작하게 해주는 규격.

HFK 맥락에서

  • 카드뉴스 HTML → 피그마 프레임으로 옮기기 (/export-to-figma)
  • 피그마 시안 → 클로드가 읽고 Imweb HTML로 변환
  • 대시보드 시안을 피그마에서 잡고 클로드가 구현

핵심 변화 — 디자인이 "이미지"에서 "AI가 읽고 쓰는 파일"로 바뀌는 중.

왜 디자이너·비개발자에게 의미 있을까

"전달"이 사라짐

예전엔 디자이너가 피그마를 만들고 개발자에게 "전달"했음. 이제 AI가 중간 번역을 하니 왕복이 실시간.

디자인이 버저닝 가능

코드와 피그마가 하나의 원본처럼 움직이면, "최신 시안이 뭔지" 문제가 줄어듦.

"Skill"이 확장됨

1회차의 Skill 개념이 이제 디자인 도구까지 호출. "카드뉴스 만들어줘" → 바로 피그마 파일까지.

교훈 — "AI로 뭘 만든다"가 텍스트/코드를 넘어 디자인 결과물까지 뻗어가는 중.

Part 3 — T 사분면
Part 3 · T 사분면 워크숍

오늘은 T, 도전할 차례입니다

지난 시간엔 "미뤄둔 일"을 시작했다면, 오늘은 "내가 못한다고 생각했던 것"을 AI에게 부탁해봅니다.

T 사분면이 어디에 있는지 다시 보기

S · Start

하고 싶었는데 미뤄왔던 일 — 습관 형성

T · Try (오늘 다룸 ★)

평소 잘 못한다고 생각했던 것 — 도전과 성장

A · Amplify

이미 잘하지만 더 잘하고 싶은 것 — 차별화

R · Recover

반복 작업을 효율화 — 시간 회수

왜 T인가? S가 "하던 일을 더 잘"이었다면, T는 "안 해봤던 일에 발을 담그는 것". AI 덕분에 진입장벽이 크게 낮아진 영역.

T = Try — 평소 못한다고 생각했던 것

"전공이 아니라서", "툴을 몰라서", "배운 적 없어서" — 그동안 손대지 못했던 일을 AI를 옆에 두고 한번 해보는 영역.

T 영역의 특징

  • 결과물이 남음 — 습관 아니라 산출물 중심
  • 첫 시도는 무조건 엉성함 — 완벽 목표로 하면 시작 못 함
  • "처음 해봅니다"를 알려주는 게 핵심
  • 한 번 뚫으면 가능 영역이 크게 넓어짐

슬기의 T 영역 사례 (실제 시도)

  • /edit-premiere-xml 영상 편집 → XML 텍스트로 편집
  • /update-card-news 디자인 → 카드뉴스 v3(a/b/c)
  • Apps Script — 프로그래밍 → 구글폼 알림, 팀추천위젯
  • /ga-report 데이터 분석 → GA4 월별 리포트
  • Growth Bot — Slack 봇 → 9종 스킬을 봇으로 노출

공통점: 전부 "이건 내 분야가 아니야"에서 시작.

T 영역 스킬은 어떻게 뚫는가

예시 1 — 카드뉴스 디자인 (비전공)

처음 명령: "인스타 카드뉴스 5장 만들어줘"

  1. v1 엉성함. 폰트 어색, 여백 이상
  2. 클로드에게 "뭐가 어색한지" 지목
  3. v2 — 정렬 틀림
  4. Figma로 직접 수정 → 스킬에 규칙 반영
  5. v3 (a/b/c) 세 스타일로 분화

예시 2 — 영상 XML 편집 (비전공)

프리미어 프로 시퀀스를 XML로 내보낸 후, 클로드가 텍스트로 편집하고 다시 임포트. 타임라인 UI 없이.

T 영역 공통 구조

  1. 맥락 고백 — "이 분야 처음입니다"
  2. 작은 목표 — "일단 한 번 돌아가는 것"
  3. 관찰 — 어디가 어색한지 구체적 지목
  4. 반복 — "이것만 고쳐보자"를 3~5번

기억할 것
T 영역의 가장 큰 장벽은 기술이 아니라 "난 못할 것 같다"는 감정. "처음입니다"를 말하는 순간 문이 열립니다.

T in Detail · 예시 3

웹캠 모션 인식으로 인터랙티브 아트 만들기

왜 이 예시가 T의 정수인가

  • "미디어 아트는 예술+공학 전공자가 하는 것"이라는 인식
  • 실제로는 HTML 한 줄 + ml5.js PoseNet이면 시작 가능
  • 그동안 진입 자체가 막혀 있던 영역 — AI로 문이 열림

결과물이 "감각적"이라 성공 경험이 오래 남음.

처음 명령 (맥락 고백)

# 맥락 고백
코딩 처음. HTML 띄우는 법만 앎.
# 작은 목표
웹캠으로 내 손 위치를 인식해서
화면에 원이 따라다니게 하기.
# 원하는 결과 형식
HTML 파일 1개, 브라우저 실행.
# 체크 방법
코드 전에 뭘 설치해야 하는지 먼저 알려줘.

진화 과정 (5번의 반복)

  1. v1 p5.js + ml5.js 연결. 콘솔에 손 좌표
  2. v2 좌표 위치에 원 그리기. "되는구나"
  3. v3 원에 색 그라데이션. 선택지 3개 요청
  4. v4 이전 위치 잔상. 물감 번지는 느낌
  5. v5 손 속도에 따라 붓 두께 변화

실제로 본인이 알아야 한 것

  • HTML 파일을 브라우저로 열 줄 아는 것
  • "카메라 권한 허용"을 누를 줄 아는 것
  • "이게 왜 이래?"를 클로드에게 물어볼 수 있는 것

p5.js도 PoseNet도 수학도 몰라도 됨.

T의 정수 — "나는 못해"에서 "이걸 내가 만들었다"까지, 한 세션 2시간이면 충분한 시대.

Workshop · Step 1

STEP 1 — 못한다고 생각했던 것 떠올리기 (5분)

"못한다"의 정의

  • "내 전공이 아니라서" 미뤄둔 것
  • "툴을 배운 적이 없어서" 시도 못 한 것
  • "전문가한테 맡겨야 하는 일"로 포기한 것
  • "나는 저런 건 못해"라고 습관적으로 말한 것

업무·취미 모두 OK (디자인·영상·코딩·데이터 분석 등).

예시 (참고용)

  • 간단한 홍보 포스터 직접 만들기
  • 엑셀 매크로로 반복 작업 자동화
  • 유튜브 영상 자르기 / 자막 넣기
  • 웹캠 모션 인식으로 인터랙티브 아트 (p5.js + 카메라)
  • 간단한 랜딩페이지 만들기
  • 설문 데이터 시각화
  • 블로그/뉴스레터 레이아웃

"미디어 아트는 전공자만"이 AI로 깨지는 중.

활동 — 노트에 3개 적어보세요. "못할 것 같은 이유"도 한 줄씩.

Workshop · Step 2

STEP 2 — 가장 가벼운 1개 고르기 (5분)

고르는 기준 3가지

  1. 실패해도 남에게 피해 안 가는 것 — 내 실험물
  2. 결과물이 한 장/한 파일로 나오는 것 — 범위가 좁을수록 좋음
  3. 검색만으로는 배우기 벅찼던 것 — 옆에서 알려줄 사람이 필요했던 영역

하지 말 것

  • "한 달 걸리는 프로젝트"는 T에 넣지 마세요
  • "전문가 수준으로 마무리" 목표 금지
  • 여러 명이 같이 봐야 하는 결과물은 보류

활동 — 3개 중 가장 작고 무해한 것에 동그라미 하나.

Workshop · Step 3

STEP 3 — 첫 명령 만들기 (10분)

T 영역 4칸 양식

  1. 맥락 고백 — "이 분야 처음입니다"
  2. 작은 목표 — "일단 한 번 돌아가는 결과 1개"
  3. 원하는 결과 형식 — "HTML / PNG / 차트 1개"
  4. 체크 방법 — "결과 보여주고 물어봐주세요"

S와의 차이: "처음 해봅니다"를 명시.

완성 예시 — "홍보 포스터 처음 만들기"

# 맥락 고백
디자인 비전공. 포스터 만들어본 적 없음.
# 작은 목표
인스타용 정사각형 1장이면 충분.
# 원하는 결과 형식
HTML 파일, 문구 3줄, 배경색 1개.
# 체크 방법
완성 전 브라우저 시안 확인.

활동 — 노트에 4칸을 그리고 각각 한 줄씩. "처음입니다"를 꼭 쓸 것.

Workshop · Step 4

STEP 4 — 실행 + 3번 피드백 (15분)

실행 방법

  1. VS Code 클로드 코드 채팅창 열기
  2. 4칸 내용을 한국어로 풀어 입력
  3. 첫 결과 받음 — "엉성할 것을 예상"
  4. 어디가 어색한지 구체적으로 1~2개 피드백
  5. 3번째 반복까지만. 그 이상은 나중에

입력 예시 (그대로 써도 됨)

> 디자인 비전공이고, 포스터를
  직접 만들어본 적 없어.
  인스타용 정사각 1장이면 충분해.
  먼저 HTML로 시안을 만들어줘.
  문구는 '4월 독서 모임 모집' 한 줄.
  시안 보고 피드백할게.

> (결과 받은 뒤)
  배경색이 너무 진해.
  글자는 가운데 정렬로.

3번째 피드백까지만. 그 이후엔 멈추고 기록.

자주 막히는 곳

  • "잘 모르겠어요" → 클로드에게 선택지 3개를 달라고 요청
  • 결과가 엉뚱함 → 맥락 고백을 더 자세히
  • 한 번에 완벽하게 하려는 유혹 → "다음 1개만 고쳐줘"로 좁히기
