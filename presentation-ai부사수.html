Introduction

클로드 코드 = 내 말을 이해하는 개발자

아무렇게나 내 마음대로 문제를 입력하면 알아서 코드를 설계해 문제를 해결해주는 AI

클로드 코드는 원래 개발자가 쓰는 도구였음. 더 효율적인 프로그램을 만드는 데 사용했음.

하지만 우리는 다르게 씀. 내 컴퓨터의 파일을 정리하고, 문서를 만들고, 쇼핑몰에 올리는 데 사용함.

"나는 기획자지만, 내 말을 모두 이해하고
문제를 해결해주는 개발자가 생긴 것."

Why 클로드 코드

ChatGPT, Gemini

같은 "200개 상품 데이터 분석해줘"를 요청했을 때의 차이임.

ChatGPT / Gemini — "알려주는" AI

  • 파일 올리면 요약해서 글로 알려줌
  • 데이터가 많으면 일부만 보고 "대략" 분석
  • 결과를 복사해서 엑셀에 다시 붙여넣어야 함
  • "답변"을 주지만 실행까지는 안 해줌

클로드 코드 — "실행하는" AI

  • 내 컴퓨터의 파일을 직접 열어서 분석
  • 200개 전부 빠짐없이 처리
  • 분석 결과를 쇼핑몰에 바로 반영까지 완료
  • "답변" 대신 "결과물"을 만들어 줌

클로드 코드엔 Claude 포함

클로드 코드에 가입하면 ChatGPT처럼 대화하는 기능이 이미 들어있음.

기존 대화 이전 가능

ChatGPT/Gemini에서 나눈 대화 내용을 옮겨올 수 있음.

구독 하나로 정리

특별히 다른 서비스만의 강점이 없다면, 클로드 코드 하나면 충분함.

# 스스로 판단하며 여러 단계를 처리
Perspective

반자동화 vs 완전 자동화

AI가 알아서 해주는 "자동화"와, 중간중간 내가 확인하는 "반자동화"는 다름.

반자동화 — 클로드 코드 방식

  • Claude가 일하다가 "이렇게 할까요?" 물어봄
  • 내가 확인하고 OK하면 다음 단계로 진행
  • 결과물을 보고 수정 요청도 가능
  • 내가 전체 흐름을 직접 점검할 수 있음

완전 자동화 — MAKE, N8N 등

  • 한 번 설정하면 내가 안 시켜도 알아서 동작
  • 예: 매주 월요일 9시에 리포트 자동 발송
  • 초기 설정이 복잡하고 유지보수에 시간 소요
  • 중간에 뭔가 바뀌면 전체가 깨질 수 있음

파트너의 관점 — 매니저로서 프로세스를 직접 점검하고 주도한다는 측면에서, 반자동화가 좀 더 안전함. 클로드 코드 하나를 야무지게 잘 쓰는 것이, 괜히 자동화까지 건드려 넓고 얕게 쓰는 것보다 낫다는 생각임.

Workflow

주간 업무를 10분에

"주간 업무 준비해줘" 한마디로 아래 4단계가 자동 진행됨. 각 단계마다 확인을 받고 넘어감.

Step 1

일정 가져오기

구글 캘린더에서 세션 날짜를 찾아 문서에 반영

Step 2

쇼핑몰 반영

변경된 문서를 쇼핑몰에 자동 업데이트

Step 3

안내 레터 작성

이번 시즌 안내 레터를 자동 작성 (선택)

Step 4

Slack 공지

팀별 채널에 세션 공지 예약 발송 (선택)

Before — 직접 하면

캘린더 확인 → 엑셀 정리 → 쇼핑몰 수정 → 레터 작성 → 슬랙 발송

약 3~4시간

After — 클로드 코드로

한마디로 실행 → 중간중간 확인만

약 10~15분

Security

AI에게 뭘 말해도 되나요?

클로드 코드는 내 컴퓨터에서 작업하지만, 내가 입력한 내용은 인터넷을 통해 Claude 서버로 전송됨. 그래서 보내는 내용을 구분해야 함.

말해도 되는 것

  • "매출 시트에 컬럼이 5개 있어" (구조만 설명)
  • "이 엑셀을 월별로 분류해줘" (작업 요청)
  • "마케팅 카피 초안 써줘" (콘텐츠 작성)
  • "경쟁사 가격 비교해줘" (공개 정보)

절대 말하면 안 되는 것

  • 고객 개인정보 (이름 + 전화번호 + 주소)
  • 회사 사내 시스템 비밀번호
  • 미공개 재무제표, 인사 정보
  • 주민번호, 카드번호, 계좌번호

비밀번호 보관 방법

Slack 토큰, 쇼핑몰 비밀번호 등은 별도 파일(.env)에 저장함. 클로드 코드가 이 파일을 읽어서 쓰지만, 대화 내용에는 비밀번호가 포함되지 않음.

비밀번호 파일 만드는 순서

  1. 클로드 코드에게 "비밀번호 보관 파일 만들어줘" 요청
  2. 생성된 파일을 직접 열어서 비밀번호 입력
  3. 클로드 코드에게 "비밀번호 파일에서 읽어서 써" 라고 요청

이렇게 하면 대화에 비밀번호가 노출되지 않음.

Philosophy

책을 사는 것만으로 나의 지식이 되지 않듯

직접 해봐야 내 것이 됨

AI부사수 외에도 여러 AI 강의를 들으면, 다른 사람의 템플릿이나 방법을 얻게 됨.

하지만 책을 사는 것만으로 나의 지식이 되지 않듯, 직접 실행해보는 것이 중요함.

다른 사람의 방법을 보고 —

  • "왜 이렇게 만들었을까?"
  • "그럼 내 업무에는 어떻게 적용할까?"

까지 생각을 확장해볼 것.

Telta 추천

  • AI 시대에 필요한 업무 역량을 진단해주는 서비스
  • AI 활용 수준을 영역별로 점수로 보여줌
  • 소개 페이지만 봐도 "우리 팀에 어떤 역량이 필요한지" 감을 잡을 수 있음

마케터인데 디자인까지 해야 할 때?

Claude + Figma 추천

  • Figma(디자인 도구)에서 "이런 디자인 만들어줘" 하면 Claude가 만들어줌
  • 클로드 코드와 Figma를 연결하면 디자인 파일을 직접 수정도 가능
  • 데이터 시트와 연동해서 사이즈 변경, 디테일 수정까지
  • 이 방법으로 만든 것이 HFK 세션 핸드아웃임

회사 보안 때문에 데이터를 못 보여주는 경우

구조만 설명하고 코드만 받기

  • "매출 시트에 날짜, 금액, 카테고리 컬럼이 있어"처럼 구조만 알려주기
  • Claude가 분석 방법을 알려주면, 내 컴퓨터에서 직접 실행(R Studio 등)
  • 실제 데이터는 AI에게 보내지 않으므로 보안 유지 가능

1회차 멤버 피드백 요약 (4L)

대부분 AI를 이미 쓰고 있었지만 "겉핥기 수준"이라고 느꼈고, 실제 업무에 깊이 있게 활용하는 방법을 알고 싶어 했음.

Liked

좋았던 것

  • 실제 업무에서 돌아가는 시연
  • "이것도 되는구나" 가능성 확인
  • 한 번 시키면 여러 곳에 동시 반영되는 것

Learned

배운 것

  • 완전 자동화보다 중간에 확인하는 반자동이 현실적
  • 내 데이터를 먼저 정리하는 게 첫걸음

Lacked

아쉬웠던 것

  • 아직 AI를 깊이 써본 경험이 부족
  • 회사 보안 환경에서 어디까지 쓸 수 있는지 불확실
  • 업무 데이터가 아직 정리 안 되어 있음

Long for

하고 싶은 것

  • 나만의 스킬 만들어보기
  • 클로드 코드 설치하고 직접 써보기
  • 매일 시간 많이 쓰는 업무부터 자동화
