[5] GitHub Pages 배포 완료 (30초)

Why Claude Code

Claude Code vs ChatGPT

같은 업무를 처리할 때, 코드를 직접 실행하는 것과 대화형 분석의 차이는 뚜렷합니다.

ChatGPT (대화형 분석)

  • 파일 업로드 → 요약/분석 결과를 텍스트로 제공
  • 데이터가 크면 샘플링하거나 잘림 발생
  • 중간 과정을 검증하기 어려움 (블랙박스)
  • 결과를 다시 수작업으로 옮겨야 함

Claude Code (코드 실행형)

  • Python을 작성하고 직접 실행 → 정확한 결과
  • 전체 데이터를 빠짐없이 처리 (샘플링 없음)
  • 코드가 남아 검증/재현 가능 (투명한 과정)
  • 결과를 파일 저장, API로 바로 반영

실전 예시 — 200개 상품 분석 → 아임웹 반영: ChatGPT는 요약만 / Claude Code는 분석 → HTML → API까지 한 번에

Perspective

기획자가 개발자의 눈을 갖게 되면

"개발자가 기획자였다면 이런 것도 했을 텐데"라는 상상이 현실이 됩니다.

데이터를 직접 다루기

  • 엑셀 대신 Python으로 정확한 분석
  • 여러 소스의 데이터를 자동으로 통합
  • 분석 → 시각화 → 리포트까지 원스톱

반복 업무를 시스템화

  • 매주 하는 작업을 스킬로 자동화
  • API를 연결해 서비스 간 데이터 동기화
  • 수작업 실수를 구조적으로 방지

아이디어를 바로 구현

  • "이런 도구가 있으면 좋겠다" → 바로 제작
  • 프로토타입을 직접 만들어 검증
  • 개발팀에 의존하지 않는 실행력

에이전트 = 반자동화 — 완전 자동화(N8N/MAKE)와 달리, 에이전트는 중간 중간 나의 확인을 받습니다. 프로세스 주도권을 유지하면서 자동화의 이점을 얻는 방식입니다.

Troubleshooting

처음 설치하는 분들을 위한 안내

설치 과정에서 에러가 나더라도 당황하지 마세요. 에러 메시지를 그대로 Claude에게 붙여넣고 물어보면 됩니다.

"command not found: claude"

설치는 됐는데 컴퓨터가 claude 위치를 모르는 상태.

해결: 터미널을 완전히 껐다가 다시 열기. 안 되면 claude doctor 실행.

"Permission denied"

설치 폴더에 쓰기 권한이 없는 상태.

해결 (Mac): 명령어 앞에 sudo 붙이기. (Windows): "관리자 권한으로 실행".

PowerShell vs CMD 혼동 (Windows)

파란 창(PowerShell)과 검은 창(CMD)은 다른 프로그램. 명령어가 다릅니다.

해결: 반드시 PowerShell(파란 창)을 사용하세요.

로그인 브라우저가 안 열림

회사 보안 정책이나 VPN이 막는 경우.

해결: VPN 끄고 재시도. 또는 표시된 URL을 직접 브라우저에 복사.

어떤 단계에서든 막히면 — 에러 메시지를 그대로 복사해서 Claude에게 붙여넣으세요. claude doctor도 스스로 문제를 진단해줍니다.

Setup

Visual Studio Code에서 Claude Code 시작하기

Step 0 — VS Code 설치

  • code.visualstudio.com 접속 → 다운로드
  • 설치 파일 실행 → 안내에 따라 설치 완료
  • 설치 후 VS Code 실행

Step 1 — Node.js 설치 확인

$ node --version
# v18 이상이면 OK
# 없으면 nodejs.org 에서 설치

Step 2 — Claude Code 설치 & 실행

$ npm install -g @anthropic-ai/claude-code
$ claude # 실행 후 브라우저에서 로그인

Step 3 — VS Code 확장 설치

  • Extensions 탭 열기 (⌘⇧X / Ctrl+Shift+X)
  • Claude Code 검색 → Anthropic 확장 설치
  • 설치 후 VS Code 사이드바에서 바로 사용

VS Code에서 쓰면 좋은 점

  • 현재 열린 파일을 Claude가 자동으로 인식
  • 코드 선택 → 바로 Claude에게 질문
  • 에러 메시지를 Claude에게 바로 전달
  • 터미널과 에디터를 동시에 사용

두번째 지능이란?

첫번째 지능 — 타고난 능력

  • 학습, 분석, 판단 등 인지적 능력
  • 경험과 훈련으로 향상되지만 한계 존재
  • AI가 빠르게 대체하기 시작한 영역

  • AI를 통해 가능해지는 새로운 실행력
  • 기획자도 개발자의 눈을 가질 수 있다
  • 미뤄뒀던 일, 엄두 못 냈던 영역에 도전

HFK AI부사수 팀의 질문 — "나에게 AI 부사수가 생긴다면, 지금 당장 무엇부터 시킬 것인가?"

Core Concept

커스텀 스킬

마크다운(.md) 파일 하나로 Claude에게 복잡한 업무 절차를 가르칠 수 있습니다.

만드는 법

  1. .claude/commands/ 폴더에 .md 파일 생성
  2. 파일 안에 업무 절차를 자연어로 작성
  3. Claude Code에서 /파일명 으로 실행

잘 만드는 팁

  • 단계별로 명확하게 절차를 기술
  • 입출력 형식과 예시를 포함
  • MCP 도구명을 명시하면 정확도 향상
  • 에이전트 스킬로 여러 스킬을 연결 가능
  • GitHub에 올려 팀과 공유 가능
Security

Gen AI 보안 리스크 관리

AI를 업무에 활용할 때 반드시 알아야 할 보안 원칙입니다.

공유해도 되는 것

  • 코드, 개발 관련 질문
  • 일반적인 업무 내용, 기획 아이디어
  • 마케팅 카피, 콘텐츠 초안

피해야 하는 것

  • 비밀번호, API 키, 인증 토큰
  • 주민등록번호, 계좌번호, 카드번호
  • 고객 개인정보 (이름+연락처+주소 조합)

1. 데이터 격리

민감 정보는 .env 파일로 분리, .gitignore로 업로드 차단

2. 대화는 휘발성

대화는 나와 Claude만 볼 수 있고, 끝나면 Claude도 잊어버림

3. 로컬 실행

Claude Code는 내 컴퓨터에서 실행. 파일 내용은 처리를 위해 Anthropic 서버로 전송되지만 저장되지 않음

토큰, 아껴 쓰는 법

토큰 = Claude가 읽고 쓰는 글자 단위. 매 질문마다 지금까지의 대화 전체를 다시 읽습니다. 대화가 길어질수록 한 번 질문에 쓰는 토큰이 눈덩이처럼 불어납니다.

1. 낡은 세션 정리 — /compact

/compact = 지금까지의 대화 내용을 요약본으로 압축하는 명령어.

  • 대화가 길어졌을 때 입력하면 과거 내용을 짧게 줄여줌
  • 주제가 바뀔 때마다 한 번씩 실행하면 효과적
  • 아예 새 대화를 시작하는 것도 방법

한 세션에서 모든 걸 하려 하지 마세요. 주제별로 나누는 게 낫습니다.

2. Claude가 읽을 범위 좁히기

Claude는 "어디를 봐야 하는지" 모르면 폴더 전체를 탐색합니다. 파일 경로와 범위를 명확히 지정하세요.

  • ✗ "이 프로젝트 개선해줘" → 수십 개 파일 탐색
  • ✓ "src/auth.ts 파일의 login 함수 수정해줘" → 파일 1-2개만 읽음

경로가 명확할수록 토큰이 줄고, 결과도 정확해집니다.

3. 기본 모델 바꾸기

기본값은 Sonnet. 복잡한 작업에만 Opus로 전환해서 사용.

/model sonnet 으로 전환

4. 작업 전: /status

큰 작업을 시작하기 전에 /status로 현재 세션 정보 확인. 대화가 길어졌다면 /compact 또는 새 세션.

5. 작업 중: /cost

/cost로 현재 세션에서 사용한 비용 확인. Pro/Max 구독이면 API 비용 없이 사용 가능.

Q&A

멤버 대표 질문

직원 AI 역량 평가하려면?

Telta 추천

  • AI 시대 필요 업무 역량을 진단하는 서비스
  • 프롬프트 설계, 데이터 분석, 자동화, 리서치 등 영역별 AI 활용 수준 측정
  • 소개 페이지에서 AX에 필요한 역량을 간접 리서치 가능

마케터인데 디자인도 혼자 해야 한다면?

Claude + Figma 추천

  • Figma에서 프롬프트 입력 → Claude로 디자인 생성
  • Claude Code와 Figma를 MCP로 연결 → 파일 직접 읽고 수정 가능
  • 데이터 시트와 Figma 연동, 사이즈·디테일 변경 가능
  • HFK 세션 핸드아웃이 이 방법으로 만든 결과물

회사 보안 때문에 데이터 분석을 못한다면?

구조만 전달 + 로컬 실행

  • 데이터셋 구조를 Claude Code에게 설명 (열 이름, 분석 맥락 등)
  • 분석 코드를 받아서 로컬에서 실행 (VS Code, RStudio 등)
  • 분석은 가능하지만 반자동화는 어려움
Philosophy

심플하게, 이것 하나만 제대로

AI 서비스가 넘쳐나는 시대

ChatGPT, Gemini, Copilot, Claude, Cursor, n8n, Make, Zapier... 매주 새로운 도구가 등장합니다.

타인의 템플릿이나 스킬을 받아도, 책을 사는 것만으로 내 지식이 되지 않듯, 실제로 실행해봐야 합니다.

"코드를 실행해보고, 왜 이 코드를 만들었을까? 그럼 나는 어떻게 해볼까? 까지 고민을 전개해보세요."

복잡하게 만들면

  • 여러 도구를 연결하느라 시간 소모
  • 하나가 바뀌면 전체가 깨짐
  • 만든 사람만 이해하는 시스템

심플하게 만들면

  • 마크다운 파일 하나로 업무 자동화
  • 누구나 읽고 수정할 수 있음
  • 작은 것부터 시작해서 점진적으로 확장
Action

지금까지 본 것을 바탕으로 — 내 워크스페이스 만들기

다양한 케이스를 봤습니다. 앞으로 내 업무에 맞는 환경을 만들어봅시다.

Step 1

반복 업무 목록화

매주/매달 반복하는 업무를 적어보기. 시간이 많이 드는 것부터 우선순위 결정

Step 2

첫 번째 스킬 만들기

.claude/commands/ 에 .md 파일 하나 생성. 절차를 자연어로 작성

Step 3

필요한 서비스 연결

내가 자주 쓰는 서비스를 MCP로 연결. Claude Code 공식 문서 참고

Step 4

스킬을 조합해 에이전트로

개별 스킬이 쌓이면 여러 스킬을 묶어 에이전트 스킬로 확장

코딩을 몰라도 됩니다. 업무 절차를 자연어로 적으면 Claude가 이해합니다. 반복되는 업무가 있다면, 그것이 바로 첫 번째 스킬 후보입니다.

HFK Philosophy

AI = 기본기  ·  HFK = 인사이트와 센스

AI만 있으면?

  • 누구나 빠르게 기본기를 갖추게 된다
  • 평균의 아웃풋이 올라간다
  • 차별화가 어려워진다
  • AI = 평범함의 도구가 될 수도 있다

AI + 인사이트/센스가 있으면?

  • 기본기를 AI로 빠르게 처리
  • 남는 시간에 인사이트와 센스를 키운다
  • 업계 지식 + AI 실행력 = 진짜 강점
  • HFK는 그 인사이트·센스를 함께 키운다

기본기를 AI로 빠르게 마스터하고, 비즈니스 인사이트와 센스를 기르는 것. 그것이 HFK와 함께하는 이유입니다.
