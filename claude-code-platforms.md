# Claude Code 플랫폼별 비교

Claude Code는 같은 도구지만 어디서 쓰느냐에 따라 할 수 있는 일이 달라집니다. 권한과 기능을 한눈에 정리했습니다.

## 한 줄 요약

| 플랫폼 | 실행 위치 | 한마디로 |
|--------|---------|---------|
| **터미널 CLI** | 내 컴퓨터 | 모든 기능. 자동화 많은 분 |
| **VS Code 확장** | 내 컴퓨터 | 에디터 안에서 바로 |
| **데스크톱 앱** | 내 컴퓨터 | 터미널 부담스러우면 GUI |
| **웹앱 (claude.ai/code)** | Anthropic 클라우드 | 설치 없이 브라우저에서 |
| **모바일 앱** | 클라우드 (웹앱 경유) | 이동 중 모니터링·지시 |
| **Cowork (Remote Control)** | 내 컴퓨터 | 데스크톱 세션을 모바일로 원격 제어 |

## 클라우드 vs 로컬, 큰 차이

### 클라우드에서 돌리면 (웹앱·모바일)

- 리포지토리에 **커밋된 파일만** 보임. 로컬에만 있는 파일·.env는 못 봄
- `~/.claude/` 사용자 메모리 못 읽음
- 로컬 MCP 서버(Slack, Imweb 등) 사용 불가
- 노트북 꺼도 작업 계속됨
- 리소스 제한: 4vCPU, 16GB RAM

### 내 컴퓨터에서 돌리면 (CLI·VS Code·데스크톱)

- 모든 파일·환경변수·MCP 다 사용
- 사용자 메모리도 다 읽음
- 컴퓨터 끄거나 네트워크 끊기면 작업 중단됨
- 리소스 제한 없음 (내 컴퓨터 사양 그대로)

## 어떤 상황에 뭘 쓸까

| 상황 | 추천 |
|------|------|
| HFK 운영 스킬 돌릴 때 (Slack/Imweb MCP 필요) | **터미널 CLI** 또는 **VS Code 확장** |
| 코드 수정하면서 변경사항 시각적으로 보기 | **VS Code 확장** |
| 터미널이 부담스러울 때 | **데스크톱 앱** |
| 설치 없이 빠르게 써보기 | **웹앱** |
| 이동 중에 진행 상황 확인·지시 | **모바일 앱** + **Cowork** |
| 노트북 꺼두고 오래 돌릴 작업 | **웹앱** (리포 기반 작업만) |

## 알아두면 좋은 점

**모바일 앱은 단독 실행 안 됨.** 웹앱이나 Cowork(데스크톱 원격 제어)와 짝지어 써야 합니다.

**Cowork 모드.** 한 세션을 여러 사람이 동시에 보며 협업하는 기능. 화면 공유 없이 같은 작업을 실시간으로 함께 볼 수 있습니다.

**MCP 서버는 어디서 돌리느냐가 중요.** 로컬 MCP(예: HFK가 쓰는 Slack/Imweb MCP)는 클라우드 환경에서 못 씀. 클라우드에서는 환경 자체에 설치된 MCP만 사용 가능.
