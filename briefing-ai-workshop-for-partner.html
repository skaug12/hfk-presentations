2. Claude Code vs ChatGPT

ChatGPT — 대화형 분석

  • 파일 업로드 후 텍스트로 요약/분석
  • 큰 데이터는 샘플링 발생
  • 중간 과정 검증 어려움
  • 결과물 수작업 필요

Claude Code — 코드 실행형

  • Python 직접 실행으로 정확한 결과
  • 전체 데이터 처리
  • 코드 투명성
  • 결과 파일 저장 · API 반영 가능

3. 네 가지 핵심 개념 — Skill · Agent · API · MCP

  • Skill — 업무 절차를 적어둔 마크다운 파일 (레시피)
  • Agent — 여러 스킬을 조합하는 매니저
  • API — 외부 서비스와 데이터를 주고받는 통로
  • MCP — API를 Claude Code에 연결하는 어댑터 (멀티탭)

4. Claude Code, 어디서 쓸 수 있나?

  • 터미널 CLI 가장 강력한 방식
  • VS Code 확장 에디터 내 시각적 비교
  • 웹 claude.ai/code 설치 없이 브라우저에서
  • 데스크톱 앱 GUI 방식

→ 플랫폼별 권한·기능 상세 비교

7. 토큰 절약 팁

  • 범위 좁히기 — "이 프로젝트 개선해줘" ✗ → "src/auth.ts의 login 함수 수정해줘" ✓
  • 주제별 세션 분리 — 한 세션에서 모든 걸 하지 말고, 주제가 바뀌면 새 대화
  • 모델 전환 — 기본값 Sonnet, 복잡한 작업만 Opus
  • /compact — 대화가 길어지면 과거 내용을 요약 압축

8. STAR 프레임워크 — 내 AI 활용 계획

S — Start

하고 싶었는데 미뤄왔던 일 → 습관 형성

T — Try

평소 잘 못한다고 생각했던 것 → 도전과 성장

A — Amplify

이미 잘하지만 더 잘하고 싶은 것 → 차별화와 전문성

R — Recover

반복적인 작업을 효율화하는 것 → 효율과 재투자

바쁜 직장인 — Start + Recover: 미뤄둔 습관 만들고, 반복 업무 줄이기

균형 추구 — Start + Try: 미뤄둔 것 시작하고, 못하던 것에 도전

STAR 축 맥락: X축은 역량·기반 (약함 → 강함), Y축은 지향 (효율 → 성장). S는 좌상단, A는 우상단, T는 좌하단, R은 우하단.

9. 강점차별화 2×2: 선호도와 인정도

같은 축 맥락으로, 내가 하는 업무를 선호도(X축)와 인정도(Y축)로 매핑.

  • Q1 강점: 좋아하고 인정받는 일. 이벤트 기획·운영·진행(100명), 파트너 섭외, 팀 기획, 웹사이트 제작. STAR의 A에 대응
  • Q2 전략적 업무: 덜 좋아하지만 인정받는 일. 이벤트 기획(25명 이하), CS. STAR의 S/T에 대응
  • Q3 축소 고려: 덜 좋아하고 덜 인정받는 일. SNS, 슬랙 운영, 영상, 매거진, TF. STAR의 R에 대응
  • Q4 숨은 강점: 좋아하지만 덜 인정받는 일. 중간 서베이, 상세페이지 제작. STAR의 T에 대응

10. BCG 매트릭스: HFK 운영에 적용하면

같은 축 맥락으로, HFK 운영 영역을 점유율(X축, 현재 강점)과 성장률(Y축, 미래 가치)로 매핑.

  • Star ★: 성장하는 시장에서 강한 포지션, 계속 투자. AI부사수 팀, 강점차별화 팀, Claude Code 스킬 생태계, 100명 이벤트
  • Question Mark: 잠재력 있으나 포지션 약함, 투자 결정 필요. AI 컨퍼런스, Figma MCP 연동, 다중 컴퓨터 환경 동기화
  • Cash Cow: 안정 수익, 효율화로 수확. 정규 성장트랙 팀, 세션 핸드아웃, 상품 운영, 시즌레터
  • Dog: 성장도 포지션도 낮음, 철수·축소. SNS 운영, 매거진, 소규모 TF

세 매트릭스 대응: Star ↔ Q1 ↔ Amplify · Question Mark ↔ Q2 ↔ Start · Cash Cow ↔ Q4 ↔ Recover · Dog ↔ Q3 ↔ Try
