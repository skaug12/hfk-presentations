2. Claude Code vs ChatGPT

ChatGPT — 대화형 분석

  • 파일 업로드 후 텍스트로 요약/분석
  • 큰 데이터는 샘플링 발생
  • 중간 과정 검증 어려움
  • 결과물 수작업 필요

Claude Code — 코드 실행형

  • Python 직접 실행으로 정확한 결과
  • 전체 데이터 처리
  • 코드 투명성
  • 결과 파일 저장 · API 반영 가능

3. 네 가지 핵심 개념 — Skill · Agent · API · MCP

  • Skill — 업무 절차를 적어둔 마크다운 파일 (레시피)
  • Agent — 여러 스킬을 조합하는 매니저
  • API — 외부 서비스와 데이터를 주고받는 통로
  • MCP — API를 Claude Code에 연결하는 어댑터 (멀티탭)

4. Claude Code, 어디서 쓸 수 있나?

  • 터미널 CLI 가장 강력한 방식
  • VS Code 확장 에디터 내 시각적 비교
  • 웹 claude.ai/code 설치 없이 브라우저에서
  • 데스크톱 앱 GUI 방식

5. 식품 데이터 MI 대시보드 — 파이프라인

프로세스

더미 데이터 생성(5,000건) → MI 분석 → HTML 대시보드 → Obsidian MD → GitHub Pages 배포

소요 시간 — 기존 2~3일 → Claude Code 약 10초

건강기능식품 +51.1% 성장 · 제로 키워드 1위 · 상위 3사 37.1% 점유

6. 자주 쓰는 명령어

  • /compact 대화 내용 압축 (토큰 절약)
  • /model sonnet 모델 전환 (기본값 Sonnet, 복잡한 작업만 Opus)
  • /status 현재 세션 정보 확인
  • /cost 세션 비용 확인
  • claude doctor 문제 자동 진단

7. 토큰 절약 팁

  • 범위 좁히기 — "이 프로젝트 개선해줘" ✗ → "src/auth.ts의 login 함수 수정해줘" ✓
  • 주제별 세션 분리 — 한 세션에서 모든 걸 하지 말고, 주제가 바뀌면 새 대화
  • 모델 전환 — 기본값 Sonnet, 복잡한 작업만 Opus
  • /compact — 대화가 길어지면 과거 내용을 요약 압축

8. STAR 프레임워크 — 내 AI 활용 계획

S — Start

하고 싶었는데 미뤄왔던 일 → 습관 형성

T — Try

평소 잘 못한다고 생각했던 것 → 도전과 성장

A — Amplify

이미 잘하지만 더 잘하고 싶은 것 → 차별화와 전문성

R — Recover

반복적인 작업을 효율화하는 것 → 효율과 재투자

바쁜 직장인 — Start + Recover: 미뤄둔 습관 만들고, 반복 업무 줄이기

균형 추구 — Start + Try: 미뤄둔 것 시작하고, 못하던 것에 도전

9. Gen AI 보안 리스크 관리

  • 공유 OK — 코드, 기획 아이디어, 마케팅 카피, 콘텐츠 초안
  • 피해야 함 — 비밀번호, API 키, 주민등록번호, 고객 개인정보
  • 민감 정보는 .env 파일로 분리
  • 파일 내용은 처리를 위해 Anthropic 서버로 전송되지만 저장되지 않음
