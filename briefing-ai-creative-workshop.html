2. 오늘 본 4개 케이스

Case 1 — HFK 커뮤니티 운영

오프라인 학습 커뮤니티 운영을 52개 스킬로 자동화. 상품 관리·콘텐츠 생성·일정 소통·멤버 리포트·에이전트·이미지 분석까지 6개 카테고리로 정리.

Case 2 — 시즌 오픈 자동화

/run-season-ops 하나로 일정 업데이트 → 아임웹 동기화 → 시즌레터 → Slack 공지 예약까지 한 흐름. 기존 3~4시간이 10~15분으로.

Case 3 — 강점차별화 2×2

내 일을 선호도(X) × 인정도(Y)로 매핑. Q1 핵심 강점은 반복을 위임, Q3 축소 영역은 자동화·통합. 사분면마다 어떤 스킬을 붙일지 결정.

Case 4 — BCG 매트릭스

HFK 운영 영역을 점유율(X) × 성장률(Y)로 매핑. Star는 계속 투자, Cash Cow는 효율화로 수확, Question Mark는 실험으로 검증, Dog는 자동화·통합.

3. STAR 프레임워크 — 내 AI 활용 계획

S — Start

하고 싶었는데 미뤄왔던 일 → 습관 형성

T — Try

평소 잘 못한다고 생각했던 것 → 도전과 성장

A — Amplify

이미 잘하지만 더 잘하고 싶은 것 → 차별화와 전문성

R — Recover

반복적인 작업을 효율화하는 것 → 효율과 재투자

바쁜 직장인 — Start + Recover: 미뤄둔 습관 만들고, 반복 업무 줄이기

균형 추구 — Start + Try: 미뤄둔 것 시작하고, 못하던 것에 도전

4. 추천 공통 과제 — 메모에서 투두까지

01그동안의 메모를 정리하기

회의록·스크랩·아이디어 메모를 한 폴더로 모아 Claude에게 정리 부탁. 스크린샷이 많다면 /process-screenshots로 OCR.

02유의미한 인사이트 뽑기

정리된 노트에서 반복되는 키워드·패턴을 찾아 의사결정에 쓸 한 줄로 요약.

"이 메모에서 자주 등장한 주제 3가지는?"

03액션플랜 세우기

인사이트를 STAR로 분류하고, 한 분기 안에 끝낼 수 있는 단위로 쪼개기.

"이걸 S/T/A/R로 분류해줘. 분기 안에 끝낼 수 있게 쪼개줘."

04투두리스트 저장하기

액션을 Apple 미리알림 또는 Things에 마감일과 함께 자동 저장. /add-reminder로 일괄 등록.

메모 → 인사이트 → 액션 → 투두 한 흐름이 끝까지 돌면, 다음 시즌부터는 같은 절차를 스킬로 재실행하기만 하면 됩니다.
