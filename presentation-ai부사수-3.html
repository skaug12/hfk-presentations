News 1 · Project Glasswing

비개발자에게는 어떤 의미인가

평소 우리가 쓰는 거의 모든 것이 영향권

  • 운영체제 — macOS, Windows, Linux 모두 점검 대상
  • 웹 브라우저 — Chrome, Safari, Edge 등
  • 금융 시스템 — JPMorgan 같은 은행도 참여
  • 병원·전기·교통 같은 사회 기반 시스템

한 해 사이버 범죄 피해액 추정: 약 5,000억 달러.

두 가지 시사점

  1. "대단한 코드"는 사람만 짤 수 있다는 가정이 흔들림 — AI는 이미 27년 묵은 결함을 분 단위로 찾음. 비개발자도 같은 도구로 일할 수 있다는 뜻.
  2. 대기업·소규모가 같은 무기를 갖게 됨 — 예전엔 보안팀을 꾸린 회사만 안전했지만, 이제 1인 오픈소스 운영자도 무료 크레딧으로 점검 가능.

우리에게 주는 신호 — AI가 "전문가만의 영역"을 빠르게 무너뜨리는 중. 우리도 두려워 말고 도구를 켜보면 됨.

News 2 · 비개발자용 가이드

"AI로 카드뉴스 자동 생성하기"

국내에서 공개된 비개발자용 카드뉴스 자동화 가이드. Claude Desktop 앱과 Skill만으로 5장짜리 인스타 카드뉴스를 만드는 법을 정리.

카드뉴스 5장 공식

  1. HOOK — 썸네일 역할, 스크롤 멈추게
  2. PROBLEM — 문제 제기, 공감 유도
  3. LIST — 저장하고 싶게 만드는 번호 리스트
  4. STAT — 공유하고 싶은 통계
  5. CTA — 팔로우/링크 클릭/DM 행동 유도

필요한 것은 단 2가지

  • Claude Desktop 앱 (claude.com/download)
  • Claude Pro 구독 (월 $20)

실제 사용 흐름 (4단계)

  1. Step A. 앱 설치 후 "Cowork" 탭 찾기
  2. Step B. SKILL.md 파일을 첨부해서 "이 파일을 Skill로 등록해줘" 입력
  3. Step C. 양식에 주제·후킹 문장·CTA만 채워 넣고 실행
  4. Step D. 결과 HTML을 Figma "HTML to Design" 플러그인으로 옮겨서 PNG 5장 추출

중요한 장치 — HUMAN CHECK

3단계마다 사람이 내용을 확인하고 다음 단계로 넘어갈지 결정. 완전 자동화가 아니라 반자동화를 유지.

1회차에서 말한 "매니저로서 점검하며 진행"의 정확한 사례.

News 2 · 의미

왜 이 가이드가 의미 있을까

2회차 카드뉴스 Q&A의 답

지난 시간 "카드뉴스 딸깍 대량생산" 질문에 대한 실제 사례. 양식 + Skill + Figma 연동으로 같은 흐름이 가능함을 보여줌.

"Skill"이라는 단어가 익숙해질 때

Skill = 정해진 순서대로 실행되는 레시피. 1회차에서 다룬 개념이 외부 가이드에서도 그대로 등장.

한 번 만들면 계속 쓸 수 있음.

코드 한 줄 없이 가능

터미널이나 VS Code 없이 Claude Desktop 앱 안에서 모든 단계가 끝남. 가장 진입장벽이 낮은 형태.

교훈 — "Skill을 만든다"는 행위가 점점 일반인의 영역으로 내려오는 중. 우리도 오늘 작은 Skill 하나를 만들어 봅니다.

News 3 · DESIGN.md

바이브 코딩으로 디자인 시스템 활용하기

국내 디자인 스튜디오 fieldby가 공개한 "DESIGN.md" 문서. 디자인 가이드를 마크다운 한 파일로 정리해 클로드 코드가 항상 참고하게 만든 사례.

아이디어의 핵심

  • 색상, 타이포그래피, 컴포넌트 규칙을 마크다운 한 파일로 정리
  • 클로드 코드는 작업할 때마다 이 파일을 먼저 읽고 결과물을 만듦
  • 매번 "우리 브랜드 컬러는 #...", "버튼 모양은 ..."을 다시 알려줄 필요가 없음

개발자들이 README.md를 쓰는 방식을 디자이너가 가져온 것.

왜 이게 새로운가

기존엔 디자이너의 지식이 사람 머릿속이나 PDF 가이드라인에 갇혀 있었음. DESIGN.md는 그 지식을 AI가 매번 자동으로 참고할 수 있는 형태로 바꿈.

생각해볼 점
내 업무에도 "매번 같은 설명을 반복하는 것"이 있다면, 그것을 md 파일 한 장으로 정리해두는 것만으로도 큰 변화가 시작됩니다.

※ 본 슬라이드의 세부 내용은 fieldby DESIGN.md 원문을 기준으로 발표 시 보강 예정.

왜 S부터? 제일 부담 없는 칸이기 때문. "잘해야 한다"는 생각 없이 일단 시작하는 것이 핵심.

S in Detail

S = Start — 미뤄왔던 일을 시작하는 칸

"매일 하면 좋은데 자꾸 미루는 것들" — AI에게 그 일을 매일 옆에서 도와달라고 부탁하는 영역.

S 영역의 특징

  • 실패해도 부담이 적음 — 어차피 안 하던 일이니까
  • 성과보다 빈도가 중요 — 매일 짧게 하는 게 성공
  • 결과물보다 트리거가 핵심 — "AI가 먼저 말 걸어주는 구조"가 절반
  • 나를 위한 일 — 회사 KPI가 아니라 내 삶의 작은 루틴

슬기의 S 영역 4개 (실제 운영 중)

  • /hello-world — 아침에 캘린더·미리알림·어제 잔여 작업을 한 화면에 띄워주는 브리핑
  • /daily — 하루 끝에 그날 작업을 자동으로 일지로 정리
  • /cleanup-home — 흩어진 다운로드 파일을 자동 분류
  • /backup-notes — 노트를 주기적으로 백업

공통점: 전부 "매일 해야 하는 줄 아는데 자꾸 까먹는 것"이었음.

S in Detail

S 영역 스킬은 어떻게 작동하는가

예시 1 — /hello-world

아침에 명령 한 줄 입력하면:

  1. 인증 파일이 제대로 있는지 확인
  2. 오늘의 캘린더 일정 가져옴
  3. 미리알림 앱에서 오늘/늦은 항목 가져옴
  4. 어제 작업 일지에서 미완료 항목 확인
  5. 한 화면에 정리해서 보여줌

예시 2 — /cleanup-home

홈/다운로드 폴더를 훑고, 파일 종류·날짜로 자동 분류 폴더에 이동. "이거 어디에 둘까"를 매번 고민하지 않아도 됨.

공통 구조

  1. 트리거 — 언제 시작할지 (아침에, 하루 끝에, 매주 일요일에)
  2. 데이터 소스 — 어디서 정보를 가져올지 (캘린더, 폴더, 슬랙)
  3. 처리 — 어떻게 정리할지 (분류, 요약, 정렬)
  4. 결과 형식 — 어디로 보여줄지 (화면, 파일, DM)

기억할 것
좋은 S 스킬은 "내가 명령하지 않아도 알아서 말 걸어주는 구조"까지 만드는 것. 일단 오늘은 그 직전 단계인 "명령 한 줄로 시작되는 단계"까지 가봅니다.

STEP 4 · 15분

실제 실행

클로드 코드 창에 그대로 붙여 넣고 결과 확인. 막히면 같이 풀어봅니다.

Workshop · Step 1

STEP 1 — 미뤄둔 일 떠올리기 (5분)

"미뤄둔 일"의 정의

  • "매일 하면 좋은데 안 하고 있는 것"
  • "한 번 시작하면 좋은데 시작이 안 되는 것"
  • "머릿속엔 있는데 손이 안 가는 것"

업무도 좋고, 개인 영역(독서·운동·가계부)도 좋음.

예시 (참고용)

  • 매일 아침 어제 한 일 한 줄로 정리하기
  • 주말마다 다운로드 폴더 정리
  • 읽고 싶은 기사 모아두고 매일 한 개씩 요약
  • 매주 월요일 한 주 캘린더 미리 보기
  • 가계부에 어제 결제 내역 기록
  • 슬랙 중요 메시지 모아두기

활동 — 노트를 펴고 3개를 적어보세요. 잘 안 떠오르면 위 예시에서 가장 비슷한 것을 골라도 됩니다.

Workshop · Step 2

STEP 2 — 가장 가벼운 1개 고르기 (5분)

고르는 기준 3가지

  1. 5분 안에 끝나는 것 — 거창한 건 금지
  2. 결과를 누가 검사 안 하는 것 — 회사 KPI 말고 내 삶의 일
  3. 매일 또는 매주 반복되는 것 — 일회성 작업은 R 영역으로

하지 말 것

  • "내 인생을 바꿀 거대한 습관" 같은 거 고르지 마세요
  • "AI가 알아서 다 해주는" 같은 막연한 기대 금지
  • 다른 사람과 협업이 필요한 것은 보류

활동 — 3개 중 가장 작고 가벼운 것에 동그라미 하나. 그것이 오늘 우리가 만들 S 스킬의 주제입니다.

Workshop · Step 3

STEP 3 — 4칸으로 명령 만들기 (10분)

4칸 양식

  1. 트리거 — "언제 이 일을 할 것인가?"
  2. 소스 — "어디서 정보를 가져올 것인가?"
  3. 처리 — "어떻게 가공할 것인가?"
  4. 결과 — "결과를 어디로 보내줄 것인가?"

한 칸당 한 줄이면 충분합니다. 완벽할 필요 없음.

완성 예시 — "매일 캘린더 정리"

# 트리거
매일 아침 9시
# 소스
구글 캘린더 오늘 일정
# 처리
시간순 정렬, 회의 / 개인일 분류
# 결과
메모 앱에 한 장으로 정리

활동 — 노트 한 장에 4칸을 그리고 각각 한 줄씩 채워보세요. 옆사람과 비교해도 좋습니다.

Workshop · Step 4

STEP 4 — 실제로 실행해보기 (15분)

실행 방법

  1. VS Code에서 클로드 코드 채팅창 열기
  2. 4칸 내용을 그대로 한국어 문장으로 풀어 입력
  3. 클로드가 묻는 질문에 한 줄씩 대답
  4. 결과 확인 — 잘 되면 다음 칸 추가, 안 되면 막힌 부분 같이 풀기

자주 막히는 곳

  • "권한이 없다"는 메시지 → 캘린더/슬랙 등 인증 필요
  • "파일을 찾을 수 없다" → 정확한 폴더 경로 알려주기
  • 너무 많은 걸 한 번에 시키기 → 한 단계씩 나누기

입력 예시 (그대로 써도 됨)

> 매일 아침 9시에
  구글 캘린더 오늘 일정을
  시간순으로 정리하고
  회의/개인일정으로 분류해서
  ~/오늘일정.md 파일로 저장하는
  스킬을 만들어줘.

> 먼저 한 번만 실행해서
  결과를 보여줘.

결과가 마음에 들면 그제서야 "이걸 매일 자동으로 실행해줘"라고 추가.
